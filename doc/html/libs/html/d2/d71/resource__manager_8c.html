◆ ResourceManagerAcquireResource()

rm_status_t ResourceManagerAcquireResource ( rm_resource_id_t  res,
rm_callback  callback 
)

Acquire ownership of a resource.

If the resource is currently owned by another module then that module will be notified, via its callback, and it will either release the resource or keep it.

If the other module keeps the resource this module will be queued and notified when the resource becomes free again.

A maximum of one module can be queued for a resource. Any more acquire requests for that will be rejected immediately.

Parameters
resResource to request access to.
callbackCallback used to notify caller when resource status changes.
Returns
rm_status_t rm_status_success if the resource was allocated to the caller, an error code otherwise.

void ResourceManagerReleaseResource ( rm_resource_id_t  res,
rm_callback  callback 
)

Release ownerdhip of a resource.

Release ownership of a resource. If another module is waiting for the resource it will be notified via its callback.

Releasing a resource must only be done once you have released all system resources it was using (e.g. disconnecting and closing a Source for a microphone).

Parameters
resResource to request access to.
callbackCallback used when reserving the resource.
