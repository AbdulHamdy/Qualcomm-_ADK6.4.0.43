Detailed Description

DESCRIPTION
This file defines all APIs for the Message Notification Service (MNS) Initialization and Connection management. The library allows only one active application to wait for incoming MNS connections from the remote device. It does not block multiple MNS connect request from the same MSE device. It is the applications responsibility to accept only 1 MNS connection per device for specification compliance and limit the maximum number of MSE remote devices for optimal resource usage.

bool  regStatus

Register or Unregister the message notifications.

Parameters
masSessionThe session handle of the MAS session.
regStatusTRUE to register and FALSE to unregister.

This API allows the application to set the Notification mode on the MSE side. The application must set the device to connectable mode and start the MNS service by calling MapcMnsStart() before turning ON the notification. The application may shutdown the MNS service by calling MapcMnsShutdown() if the Notification mode is turned OFF on all connected MAS sessions or no MAS connections are available.

Returns
Message MAPC_MAS_SET_NOTIFICATION_CFM is sent to the application.

Accept or reject the incoming MNS Connection.

Parameters

addrThe Bluetooth device address of the remote MSE device.

The application must call this API on receiving the message MAPC_MNS_CONNECT_IND from the library for incoming connections. The library returns one or more MAPC_MNS_CONNECT_CFM as a result of this API Call. After establishing a session, the library notifies MAPC_MNS_CONNECT_CFM to the application with mapc_pending. After a successful MNS Connection, it notifies MAPC_MNS_CONNECT_CFM with mapc_success.

Shutdown the MNS Service.

Parameters
recordHandle- The SDP record handle of the MNS Service.

The application can use this API to stop the MNS service. This API will not disconnect the existing MNS connections, but it will not accept any new MNS connections after calling this API. This API unregister the SDP record and deallocate the MNS server channel.

void MapcMnsStart ( Task  theAppTask,
bool  popSdpRec,
uint8  mnsChannel 
)

Start the Message Notification Service.

Parameters
theAppTaskThe current application task. The library notifies this task on receiving any incoming MNS connection request from the MSE device.
popSdpRecSet this flag to TRUE to register a default SDP record using a default RFCOMM server channel. Otherwise the library expects the application to register the SDP record after calling this API.
mnsChannelRFCOMM Server channel to be allocated for MNS. If popSdpRec is set to TRUE, the library ignores this value and registers a default MAP Server channel allocated by the MAP library. Otherwise, this must be matching with the RFCOMM Server channel in the application SDP record.

The application can start only one Message Notification Service. For better utilization of resources, the application must turn ON the Message Notification Service just before turning on the Register Notification by calling MapMnsSetRegisterNotification(). The expected return status values are mapc_success on successfully starting the service, mapc_mns_started if the service is already running and mapc_failure if the library is unable to start the notification service.

Returns
Message MAPC_MNS_START_CFM is sent to the application.
