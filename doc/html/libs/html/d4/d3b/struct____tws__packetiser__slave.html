◆ excess_claimed

uint32 __tws_packetiser_slave::excess_claimed

Occasionally, a packet with a new ttp will received having previously received incomplete fragments of another frame. The fragments will be written in claimed space in the sink. The new fragment will need to overwrite the existing data in the claiming space. This variable is used to track the amount of pre-claimed data in the sink
