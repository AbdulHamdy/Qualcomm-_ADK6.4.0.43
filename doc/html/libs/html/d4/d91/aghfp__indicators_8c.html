Respond to AGHFP_CALL_INDICATIONS_STATUS_REQUEST_IND.

Parameters
aghfpA pointer to the profile instance which is to be disconnected.
availabilityIndicates whether network is available or not
call_statusIndicates whether there is an active call in place or not
call_setup_statusIndicates if a call setup is in progress or not (and in what direction)
call_held_statusIndicates whether a call is on hold or not
signalIndicates the network signal strength
roam_statusIndicates the roam status
battIndicates the AG battery level

No message will be passed back to the app as a result of this function.

◆ AghfpSendCallWaitingNotification()

void AghfpSendCallWaitingNotification ( AGHFP *  aghfp,
uint8  type_number,
uint16  size_number,
const uint8 *  number,
uint16  size_string,
const uint8 *  string 
)

Tell the HF that there is a call waiting, while this is already an incoming call.

Parameters
aghfpA pointer to the profile instance.
type_numberThe type of number being sent. Permitted values are 128 to 175 - see HFP 1.5 spec.
size_numberThe number of characters in the phone number
numberThe phone number of the waiting call. This is a uint8 array containing one digit of the phone number per array element. The digits are in ASCII format and may include special characters such as '+'. The number should NOT be NULL terminated.
size_stringThe number of characters in the string. (Not including any null terminator). Should be zero if string is NULL.
stringA friendly name to go with the phone number. Maybe NULL.

The application will receive an AGHFP_SEND_CALL_WAITING_NOTIFICATION_CFM message from the library to indicate the success or otherwise of the notification send attempt.
