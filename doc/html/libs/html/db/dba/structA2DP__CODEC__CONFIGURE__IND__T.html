Detailed Description

This message is sent to indicate that the application must choose how the codec should be configured, based on the remote codec capabilities. The application must respond immediately using the A2dpCodecConfigureResponse API. The application must choose the required parameters to use for this codec.

Only ever received if the application is setting up an outgoing media connection and the local SEP, as identified by the local_seid, is NOT configured to have codec parameters chosen automatically by the A2DP library.
The codec_service_caps will start with AVDTP_SERVICE_MEDIA_CODEC as the first byte, and then the other codec information follows. The format is detailed in the 'Media Codec Capabilities' section of the AVDTP profile specification.
