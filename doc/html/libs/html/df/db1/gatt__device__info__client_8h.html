◆ GDISC

typedef struct __GDISC GDISC

The Device Info internal structure for the client role.

This structure is visible to the application as it is responsible for managing resource to pass to the Device info library. The elements of this structure are only modified by the Device info library. The application SHOULD NOT modify the values at any time as it could lead to undefined behaviour.

◆ GattDeviceInfoClientDestroy()

bool GattDeviceInfoClientDestroy ( GDISC *  device_info_client)

When a GATT connection is removed, the application must remove all client service instances that were associated with the connection (using the CID value). This is the clean up routine as a result of calling the GattDeviceInfoClientInit API. That is, the GattDeviceInfoClientInit API is called when a connection is made, and the GattDeviceInfoClientDestroy is called when the connection is removed.

Parameters
device_info_clientThe client instance that was passed into the GattDeviceInfoClientInit API.
Returns
TRUE if successful, FALSE otherwise

◆ GattDeviceInfoClientInit()

bool GattDeviceInfoClientInit ( Task  app_task,
GDISC *const  device_info_client,
const GATT_DEVICE_INFO_CLIENT_INIT_PARAMS_T *const  client_init_params 
)

After the VM application has used the GATT manager library to establish a connection to a discovered BLE device in the Client role, it can discover any supported services in which it has an interest. It should then register with the relevant client service library (passing the relevant CID and handles to the service). For the Device Info client it will use this API. The GATT manager will then route notifications and indications to the correct instance of the client service library for the CID.

Parameters
app_taskThe Task that will receive the messages sent from this Device Info client library.
device_info_clientA valid area of memory that the service library can use.
client_init_paramsInitialisation parameters, CID,Start handle, end handle of Device Information Service
Returns
TRUE if successful, FALSE otherwise
