Send a message to Bluestack requesting the encryption of a data block of arbitrary length using SHA256.

Usage for SHA256 Block Encryption

The user is expected to keep track of the current index progress within each array they intend to pass as data to be hashed. This can be done by incrementing said index every time a crypto_int_success message is received for the particular task, or any other way the user deems appropriate.

The functions also expect the user to pass the length of the data array as a parameter, since there is no way for the function to calculate their size after they've been collapsed to a pointer.

Both the index and length must be in octets, despite the arrays themselves holding words. If the array contains an odd number of octets, the user can include the last octet by itself, since the API will pad it with zeroes.

Parameters
theAppTaskThe task that called this crypto function.
data_arrayAn array holding the data to be encrypted. The array can be of any length.
array_sizeThe size of the array passed as the previous argument in octets.
Returns
None. A confirmation message containing the encrypted data (if the data block was not greater than 32 bytes) or signalling interim success is returned from the Bluestack and is handled by the message handler in bluestack_handler.c.

Send a message to Bluestack requesting the continuation of the SHA256 encryption operation for the remaining data.

Parameters
theAppTaskThe task that called this crypto function.
data_arrayAn array holding the data to be encrypted. The array can be of any length. This must be the same array used in the first call to SHA256 encryption.
array_sizeThe size of the array passed as the previous argument in octets.
current_indexThe index position of the next hash block's first character in the passed array in octets. In practice, this number can only be multiples of 32.
Returns
None. A confirmation message containing the encrypted data (if the remaining data block was not greater than 32 bytes) or signalling interim success is returned from the Bluestack and is handled by the message handler in bluestack_handler.c.
