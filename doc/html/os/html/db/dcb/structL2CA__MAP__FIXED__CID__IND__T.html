Detailed Description

L2CAP fixed channel map indication.

If an application has registered interest in a fixed channel using L2CA_REGISTER_FIXED_CID_REQ_T and a peer sends data on that particular fixed channel, L2CAP will automatically bring up the necessary internal structures and assign a dynamic CID for that channel. Before the data sent on the channel is passed to the application, the L2CA_MAP_FIXED_CID_IND_T will be sent to the application so it can be identified from which device the data was sent, and to what specific fixed channel number.

Note: The connection context number (con_ctx) will not be valid before the application has sent a L2CA_MAP_FIXED_CID_RSP_T and L2CAP can not guarantee that crossovers will not happen.
