325 /****************************************************************************
326 NAME
327  buf_raw_write_only_unmap - unmap the buffer for writing ONLY.
328 
329 FUNCTION
330  This function unmaps the buffer from the write port. A buffer
331  that is mapped using any of the nbuf_raw_write_only_map_xxx functions
332  MUST be unmapped once it is done with the writing of this buffer.
333 
334  If a nested function is called after mapping a buffer but before
335  unmapping it, the nested function MUST use xxx_save_state functions
336  to map the buffer. The unmapping is done using the appropriate unmap
337  function. Additionally the buf state MUST be restored using one of the
338  appropriate restore functions. If a write update is required, it
339  can use xxx_update_restore fucntions, otherwise it can use
340  buf_raw_buf_restore_state()
341 
342  Note that the updation of the message length is unaffected by this
343  new buffer usage model
344 
345  Example usage:
346 
347  fn1
348  {
349  uint8 *p_data1;
350  ......
351  p_data1 = buf_raw_write_only_map_8bit(buffer1);
352 
353  write few bytes to the buffer1
354 
355  fn2(buffer2); // nested function
356 
357  buf_raw_write_only_unmap(buffer1);
358 
359  // This is required only if the index needs to be updated
360  buf_raw_write_update(buffer1);
361  }
362 
363  fn2(buffer2)
364  {
365  uint8 *p_data2;
366  ......
367 
368  p_data2 = buf_raw_write_only_map_8bit_save_state(buffer2);
369 
370  write few bytes to the buffer2
371 
372  buf_raw_write_only_unmap(buffer2);
373 
374  //This is required only if the index needs to be updated
375  buf_raw_write_update_restore(buffer2);
376  OR
377  // If neither index nor front needs to be updated
378  buf_raw_buf_restore_state();
379  }
380 
381  See the buf_raw_read_unmap() header for an example of a nested function
382  that copies one section of the buffer to another non-overlapped section
383  of the same buffer.
