Detailed Description

Message buffer structure This encapsulates a standard BUFFER implementation.

Field Documentation

◆ back

uint16 BUFFER_MSG_ss::back

The last entry that has not been despatched by the downstream end.

If front == back there are no messages to despatch.

◆ back_header_bytes_consumed

uint16 BUFFER_MSG_ss::back_header_bytes_consumed

The number of bytes of the message at back that have been consumed by upstream protocol modules.

This is used to adjust the returned length and pointers when the back message is accessed.

◆ behind

uint16 BUFFER_MSG_ss::behind

The last entry for which the data memory has not been freed.

If back == behind then there are no despatched messages waiting to be freed and if front == back == behind then the message ring is empty.
