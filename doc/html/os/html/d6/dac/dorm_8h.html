◆ dorm_sleep_sched()

void dorm_sleep_sched ( void  )

Sleep until the scheduler wants to wake up.

The function will return early as soon as it detects a pending background interrupt or if the chip detects activities on wake up sources configured via dorm_set_sleep_info().

This function calls sched_get_sleep_deadline() to retrieve earliest and latest wakeup time if it needs to know them.

The earliest and latest times returned by sched_get_sleep_deadline have the following requiremets:

earliest is the earliest time on the microsecond clock at which dorm_sleep should return, unless woken by the foreground. This corresponds to the earliest time of an event scheduled by timed_event_at_between().

latest is the last time on the microsecond clock at which dorm_sleep_sched() should return. As usual, this value is compared with the current time in a signed fashion: if the difference has the top bit (bit 31) set, then latest is treated as being in the past and the function returns immediately. This means that minor delays in the background scheduler are benign.

sched_get_sleep_deadline() also returns a boolean to indicate whether it has a deadline or not.

Sleep until the scheduler wants to wake up.

◆ kip_table

volatile uint16 kip_table[DORMID_COUNT]

Array storing the values passed down to dorm_sleep_info.

The elements are indexed by dormid, and the values are or's of bits given by dorm_state. Each bit indicates we can or can't go into shallow sleep.

This wastes quite a lot of bits, but the gain, that we can set the value atomically as a single word when called from each module, outweighs this.
