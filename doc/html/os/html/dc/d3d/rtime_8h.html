#define time_add (   t1,
  t2 
)    ((TIME)(t1) + (TIME)(t2))


Add two time values

Returns the sum of "t1 and "t2". See rtime_clock for notes on what this means.

Note
NOTES

Implemented as a macro, because it's trivial.

Typically, this will be used to add an INTERVAL to a TIME, returning a new TIME. Lint will complain about this if the result needs to be used as a TIME because at that point the result has become signed and it doesn't know it's OK to treat the same bits as an unsigned number to get a TIME. Because we are happy that the result always needs to be treated as a TIME whatever bits it contains (or CSR would have turned into dust long ago) we'll cast both arguments to TIME to shut it up.

Actually, adding a TIME to a TIME is exactly the one operation that doesn't make sense here! I don't know of a way to convince lint of that. When the new C++ library comes along in 2031 this will be fixed. Logically, we would then have 

TIME TIME::operator+(INTERVAL t2);
TIME INTERVAL::operator+(TIME t2);
INTERVAL INTERVAL::operator+(INTERVAL t2);
