39 /* Time manipulation macros
40  *
41  * There are two different data types at work here.
42  *
43  * One is a "system time" - this is something we could program
44  * into the timer hardware, or read back from hal_get_time().
45  * This is constrained by the hardware to be a 32-bit number, and is unsigned.
46  * Numbers outside the current 70-minute-ish epoch are meaningless.
47  * Adding two system times, or multiplying a system time by a constant,
48  * doesn't produce a sensible result.
49  *
50  * The other is a "time interval". This is a number of microseconds
51  * between two events, and (depending on ordering) can be negative.
52  * Time intervals are just numbers - they can be added, multiplied, whatever.
53  *
54  * time_add takes a system time and a time interval, and returns a system time.
55  *
56  * time_sub takes two system times, and returns a time interval.
57  * This is just a number, so it can be compared with other numbers
58  * (thresholds etc) directly.
59  */
