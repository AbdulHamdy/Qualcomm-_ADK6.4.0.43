RFCOMM Register Server Channel request.

This event allows the application to register a server channel with RFCOMM. This is a channel to which a remote device may request a connection. A protocol handle is also registered which will be used for notifying the application of the arrival of events for the given server channel.

If the loc_serv_chan_req is set to RFC_INVALID_SERV_CHANNEL then RFCOMM will assign the next available server channel number and return it in a RFC_REGISTER_CFM primitive. If loc_serv_chan_req is set to any other value within range then RFCOMM will register that server channel number (if not already allocated), or else arbitrarily choose the next available one.

There is a user specifiable context which will be echoed back in the corresponding CFM primitive.

There are currently no flag options available for this prim and thus the field should be set to 0.

Modem Status sequencing *.

A modem status command is used to convey control parameters for a specific dlci such as flow control, Modem signals, Break, between Port Emulation Entities. The modem signal is compulsory however the break signal is optional. The modem status command itself MUST be sent after opening a DLC but prior to sending any data on that DLC. It is the host's responsibility to ensure that it sends its own MSC request before commencing data transfer. NB This is handled automatically as part of the connection sequence for a RFC_CLIENT_CONNECT_REQ / RFC_SERVER_CONNECT_RSP. Thus these primitives are only to be used for additional MSC requests.
