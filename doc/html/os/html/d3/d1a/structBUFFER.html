Detailed Description

The buffer structure itself.

This encapsulates an MMU buffer.

A BUFFER has only one handle, called handle. It is used for both reading from and writing to the buffer.

◆ size_mask

uint16 BUFFER::size_mask

Mask to keep operations on indices within the power-of-two buffer size.

The capacity of the buffer is given by (size_mask + 1).

◆ tail

uint16 BUFFER::tail

The point up to which the buffer's data has been cleared (and, if appropriate, previous pages freed).

This follows index and outdex. A non-zero difference between index and tail (tested by BUF_GET_USED()) indicates that not all data has been freed.

The distinction between outdex and tail is largely for the benefit of maintaining the state of peripherals. When a peripheral is supplied with data up to a certain point, outdex is advanced. When that memory is no longer in use by the hardware, tail is advanced to the same point.
