◆ utils_SLL_remove()

void utils_SLL_remove ( utils_SLL *  s,
utils_SLLMember *  m 
)

Remove an existing member from this SLL (wherever it is in the list)

Order(n)

Note
For the avoidance of doubt removing a member from an SLL does not free any storage associated with the member. SLL membership is totally independent of storage management. The SLL has no idea if a member's storage is dynamically or statically allocated. This is a key feature of this intrusive SLL - it allows dynamic lists of statically allocated members and has no dependency on a dynamic allocator at all.
Precondition
utils_SLL_contains(s,m)
