Detailed Description

L2CAP fixed channel map request.

In order to be able to transmit data on a fixed channel, a dynamic CID must be allocated for the data path. This signal will request such a dynamic CID to be allocated for an identified fixed channel for a particular Bluetooth address. The assigned CID will be returned in the CFM signal.

Note that the application is responsible for determining whether the remote L2CAP entity and application is capable of transferring data on a fixed channel. Also notice that the L2CAP configuration used on the particular fixed channel is determined in L2CA_REGISTER_FIXED_CID_REQ_T. Once the fixed channel has been mapped to a dynamic CID, the normal L2CA_DATAWRITE_REQ_T and L2CA_DATAREAD_IND_T can be used. The channel can, however, only be disconnected indirectly when the ACL is closed!
