◆ ruleAction

enum ruleAction

Action required by the rules engine. This type is used as the return value from a rule.

Enumerator
RULE_ACTION_RUN 

Let the rule engine know that it should send the message defined for the rule

RULE_ACTION_COMPLETE 

Let the rule engine know that no message is needed, as the rule has been completed

RULE_ACTION_IGNORE 

Let the rule engine know that no message is needed, as the rule not required

RULE_ACTION_DEFER 

Let the rule engine know that no message is needed - AT THE MOMENT - but that it must reevaluate this rule when subsequent events occur.

RULE_ACTION_RUN_WITH_PARAM 

Let the rule engine know that it should send the message defined for the rule and that the message has a body with parameters.

◆ ruleStatus

enum ruleStatus

Current rule status.

Enumerator
RULE_STATUS_NOT_DONE 

rule has not been processed at all

RULE_STATUS_IGNORED 

rule has been called, but did not generated an action

RULE_STATUS_IN_PROGRESS 

rule has been called, generated an action, but has not yet been completed

RULE_STATUS_COMPLETE 

rule has been called, generated an action and completed

RULE_STATUS_DEFERRED 

rule has been called, did not generate an action at this time due to incomplete rule preconditions, the rule must be reevaluated when subsequent events occur.
