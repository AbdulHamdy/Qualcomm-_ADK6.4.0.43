Stores incoming event in the sink event queue.

DESCRIPTION
Checks if incoming event has audio prompt/tone attached if yes then creates sink event queue (if not present) and then stores event in it after checking if event is to be stored or not.If event is already present then previous occurence of that event will be cancled and it will be added in the last of the queue as it is recent event.
