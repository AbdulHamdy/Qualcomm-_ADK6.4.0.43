DESCRIPTION
Perform any steps that are needed after broadcast audio input has been disconnected.
This is called immediately after broadcast audio input has been un-routed. The reason for un-routing is not given but common reasons are:
  • A higher priority audio source was routed.
  • Broadcasting was stopped.
Returns
bool TRUE if connect_params was populated successfully, FALSE otherwise.
